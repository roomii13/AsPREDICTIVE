# Encuadre regulatorio inicial

## Cómo presentar el sistema frente a ANAC / SSP / PNSO

El sistema debe presentarse como:

- herramienta de apoyo a la gestión de seguridad operacional
- tablero de priorización y alerta temprana
- apoyo a análisis de tendencias y gestión de riesgo
- soporte para trazabilidad y registro de eventos

No debe presentarse como:

- sustituto de investigación oficial
- sistema autónomo de decisión operacional
- fuente única para determinaciones regulatorias

## Uso recomendado

- Priorizar incidentes y reportes
- Detectar patrones repetitivos
- Proveer una explicación inicial del riesgo
- Apoyar reuniones de seguridad operacional
- Consolidar información para SMS/SSP interno

## Recomendación institucional

Para una oferta formal en aeropuerto u operador:

1. presentar el sistema como capa de apoyo al SMS
2. aclarar que la IA sugiere y no reemplaza juicio experto
3. documentar trazabilidad de datos, alertas y auditoría
4. incluir responsable humano de validación
